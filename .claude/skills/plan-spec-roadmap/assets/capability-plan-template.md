## Definition of Ready

- [ ] Objetivo, escopo, atores e dependências definidos
- [ ] Increments verticais, changes e capability specs mapeados
- [ ] Exatamente um incremento selecionado e `READY_FOR_SPEC` para o próximo prompt
- [ ] Nenhuma decisão `CRITICAL/OPEN`
- [ ] Regras, falhas e critérios de aceitação verificáveis
- [ ] Aspectos técnicos e não funcionais definidos ou não aplicáveis
- [ ] Diagrama sincronizado ou explicitamente não aplicável

## Definition of Done

- [ ] Implementação e desvios rastreados
- [ ] Testes e evidências vinculados aos critérios
- [ ] Plano, spec, código e documentação reconciliados
- [ ] Revisão e arquivamento autorizados
